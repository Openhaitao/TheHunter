## 1. 产品定位（一句话）

**浏览任何网页时，一键把页面上的信息按你定义的模板结构化，存进你指定的飞书多维表格。**

LinkedIn 只是第一个「猎场」。产品本体是一个通用壳：不写死任何网站、任何表结构，
一切由用户配置的「捕获模板」驱动。

## 2. 核心概念：捕获模板（Recipe）

一个 Recipe 定义"在哪抓、抓什么、存到哪"，是整个产品唯一的核心对象：

| 组成 | 内容 | 示例（LinkedIn 人才 Recipe） |
|---|---|---|
| 适用站点 | URL 匹配规则（域名/路径通配） | `linkedin.com/in/*` |
| 抓取方式 | 默认通用正文提取；可选站点定制选择器 | LinkedIn 定制：姓名/头衔/About 区块 |
| 结构化 Schema | 字段清单：名称、类型、提取说明（给 LLM 的指令） | 姓名 / 当前公司 / 职位 / 地区 / 亮点 |
| 输出目标 | 飞书表格地址 + 字段映射 | 「公司人员名单」表，姓名→姓名列… |
| 去重键 | 用哪个字段判断记录已存在 | LinkedIn 链接 |

内置 1~2 个官方预设（LinkedIn 个人页 → 你的人才表），用户可以复制预设改出自己的模板，
也可以从零新建（比如：产品官网 → 竞品追踪表；新闻页 → 行业情报表）。

## 3. 用户流程

### 配置（一次性，options 页）
1. 新建 Recipe → 填站点规则
2. 定义字段：手动添加，或 **「从飞书表格导入」**——填目标表地址后自动拉取列名/类型生成 Schema（P1）
3. 绑定输出：粘贴表格 webhook 地址（v1）或选择授权后的表（P1）
4. 保存，即刻生效

### 日常使用（每次 10 秒）
1. 浏览到目标页面，点击工具栏图标
2. 插件自动匹配 Recipe（多个匹配则让用户选）
3. 抓取正文 → LLM 按 Schema 输出结构化 JSON
4. **预览卡片**：逐字段展示结果，可手动修正、可标记某字段留空
5. 点「保存」→ 写入飞书；命中去重键则提示「已存在，是否更新/跳过」

## 4. 技术架构（分四层，全部配置驱动）

```
┌─ 抓取层 ─────────┐  通用：Readability 式正文提取（任何网站可用）
│                  │  定制：Recipe 可附加 CSS 选择器（LinkedIn 等预设自带）
├─ 结构化层 ───────┤  LLM（硅基流动，沿用现有 options 里的 key）
│                  │  按 Recipe Schema 生成 JSON Schema 约束输出
├─ 映射层 ─────────┤  LLM 字段 → 飞书列的映射与类型转换（文本/链接/多选/日期）
├─ 同步层 ─────────┤  v1: Base webhook（零授权，现有能力）
│                  │  v2: 飞书开放 API（支持查重、更新、读回表结构）
└──────────────────┘
```

现有代码的复用度：LinkedIn 抓取、LLM 结构化、webhook 推送三块都已存在，
本方案主要是把"写死的一条链路"重构成"Recipe 驱动的 N 条链路"。

## 5. 分期计划

### P0 — 通用壳 MVP
- Recipe 数据模型 + chrome.storage 持久化
- options 页：Recipe 列表 / 新建 / 编辑 / 删除（字段手动定义）
- 通用正文抓取 + LLM 按 Schema 结构化
- 预览-修正-保存流程；webhook 写入
- 内置 LinkedIn 预设（对齐你链接里的那张表）

### P1 — 顺手度
- 「从飞书表格导入字段」：授权飞书自建应用后自动生成 Schema + 映射
- 去重键查重（需开放 API）：已存在提示更新/跳过
- 列表页批量捕获（如 LinkedIn 搜索结果页一次抓多人）

### P2 — 生态
- Recipe 导入/导出/分享（JSON 文件）
- 多输出目标：Notion、Excel/CSV 下载
- 右键菜单「捕获选中区域」、快捷键
